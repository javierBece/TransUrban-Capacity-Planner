# Guía de Contribución

⚠️ **Nota Importante:** Este es un **proyecto académico finalizado en abril de 2026** que **NO está en desarrollo activo**. 

Sin embargo, valoramos tu interés y respeto por el trabajo realizado. Este documento proporciona pautas si deseas:
- Reportar bugs encontrados
- Sugerir mejoras (aunque no se garantiza implementación)
- Usar como base para tu propio proyecto
- Contribuir mejoras puntuales

## 📋 Código de Conducta

Por favor, sé respetuoso con otros contribuidores y con los autores originales. Cualquier abuso será reportado a los mantenedores del proyecto.

## 🚀 Cómo Contribuir

### Reportar Bugs

Antes de crear un reporte de bug, verifica la lista de issues para asegurarte de que el problema no ha sido reportado previamente. Si encuentras un bug:

1. **Usa un título descriptivo** para el issue
2. **Describe los pasos exactos** para reproducir el problema
3. **Proporciona ejemplos específicos** para demostrar los pasos
4. **Describe el comportamiento observado** y explica cuál es el problema
5. **Incluye capturas de pantalla** si es posible
6. **Menciona tu entorno:** Python version, SO, etc.

### Sugerencias de Mejora (Enhancements)

Las sugerencias de mejora son bienvenidas. Crea un issue con:

1. **Un título claro**
2. **Descripción detallada** del comportamiento esperado
3. **Justificación** de por qué esta mejora sería útil
4. **Ejemplos** de implementaciones en otros proyectos

### Pull Requests

**⚠️ Nota sobre PR:** Como el proyecto no está en desarrollo activo, la revisión y merge de PRs puede tomar tiempo. No se garantiza que todos los PRs sean aceptados, pero evaluaremos contribuciones de calidad que mantengan la integridad del proyecto.

1. Fork el repositorio y crea una rama desde `main`
2. Si has añadido código, agrega tests
3. Asegúrate de que el código siga el estilo del proyecto
4. Escribe un mensaje de commit claro y conciso
5. Abre un Pull Request con descripción de los cambios

## 📝 Guía de Estilo

- Sigue [PEP 8](https://www.python.org/dev/peps/pep-0008/) para Python
- Usa nombres descriptivos para variables y funciones
- Comenta código complejo
- Mantén líneas de menos de 100 caracteres

## 🧪 Testing

Antes de hacer un PR, asegúrate de:

1. Probar tu cambio localmente
2. Verificar que no rompas funcionalidad existente
3. Agregar tests para nueva funcionalidad

## 📦 Requisitos de Commit

- Mensajes claros y descriptivos en español o inglés
- Agrupa cambios relacionados
- Referencia issues cuando sea relevante (e.g., "Closes #123")
